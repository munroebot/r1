2.0 - Syntax:

void createCalendarWidget(string widget_id,
                          string date_format,
                          string EDIT | NO_EDIT,
                          string ICON | NO_ICON,
                          string icon_path)
string widget_id
The widget_id sets the values for the text field <input> tag name and id attributes.
string date_format
The date_format sets the date format (currently yyyy/mm/dd, mm/dd/yyyy or dd/mm/yyyy)
string EDIT | NO_EDIT
The EDIT | NO_EDIT string argument tells the widget to appear in read/write or read-only mode.
string ICON | NO_ICON
The ICON | NO_ICON string argument causes the widget to display a 'calendar' icon next to the text field.
string icon_path
The icon_path is a relative or absolute path to the icon that should be displayed if the ICON argument is set.

3.0 - Usage Example:

To use the widget, you have to include a javascript library and a cascading style sheet, as shown below. Standard convention is to place these in the <head> section of the document. Be sure to change the href and src attributes to reflect your setup. 

<link type="text/css" rel="stylesheet" href="calendar.css" />
<script type="text/javascript" src="calendar.js"></script>

Where ever you want to place a widget, use the following code: 

<script type="text/javascript">
    createCalendarWidget('MgrDueDate','mm/dd/yyyy','EDIT', 'ICON','./images/calendar-icon.gif');
</script>
Important!

You will need to set the widget_id ('MgrDueDate' in the above example) to something unique, as you would when creating a text field. The widget_id controls the name and id attributes of the date entry field <input> tag .

4.0 - Cascading Style Sheet Guidelines

The widget was designed with flexability in mind, so everything has been styled using CSS. You can modify the look and feel of the widget by modifying the entities in the calendar.css file.
